Implementation plan · Capacitor plugin

Rework: AdMob Next-Gen → Unity LevelPlay

Scope Migrate the Capacitor ad plugin off Google AdMob onto Unity LevelPlay mediation 9.4.0. The plugin wraps LevelPlay — never a single network. Mediation networks are opt-in and install-time configurable (default: none). Add impression-level revenue events. Android is a rewrite; iOS is built fresh from an echo stub.

Banner · Interstitial · Rewarded App Open & Rewarded Interstitial dropped
The one rule. The plugin core talks only to LevelPlay. No mediated network's SDK is ever imported or called directly — not even for privacy. Meta / AppLovin / Unity Ads etc. are interchangeable opt-in adapters behind LevelPlay.
Key facts driving the design the "why" behind every decision
  • LevelPlay is the integration surface; each network is an adapter. The app talks to LevelPlay ad-unit IDs from the dashboard; LevelPlay abstracts each network's own placement IDs — so adUnitId stays, no rename to placementId. The core SDK carries ironSource's own first-party demand (iSX programmatic), so it serves ads with zero adapters. (ironSource direct-demand campaigns ended 2026-04-30; iSX continues.)
  • Adding a network = two layers. (1) Adapter — a native dependency baked into the binary at build time. (2) Dashboard — enabling the network + credentials per ad unit. Layer 1 can't be a runtime flag, so it is install-time-configurable; layer 2 needs no rebuild. initialize only takes the app key.
  • No self-managed preload pool. A LevelPlay ad object is reusable: construct once, loadAd() → isAdReady() → showAd() → loadAd() again. The old ArrayDeque buffer / pollAndShow API collapses into plain load* / is*Ready / show*.
  • Async init is mandatory. Ad objects may only be created after onInitSuccess.
  • Platform class names diverge. Android LevelPlay* prefix; iOS LPM*. The TypeScript layer normalizes both behind one API.
  • An adapter may not pull its network SDK (Android). Native Gradle does not guarantee transitive resolution; the install hook declares adapter + network SDK as a pair per entry in its name→coordinates table.
  • Typed 9.4.0 privacy API — no legacy calls. setGDPRConsents(Map) (per-network, replace-all), setCCPA(boolean), setCOPPA(boolean). Deprecated setConsent / do_not_sell / is_child_directed are not used. Only string flag the core sets: setMetaData("is_test_suite","enable").
  • All privacy goes through LevelPlay. It forwards GDPR / CCPA / COPPA + the advertiser-tracking flag to its adapters. No per-network SDK call, no NetworkPrivacyBridge, no reflection guard.
  • iOS ATT is the app's job. LevelPlay does not show the ATT prompt. The plugin triggers requestTrackingAuthorization before init and forwards the result to LevelPlay's advertiser-tracking API.
  • No testMode init flag. Removed in LevelPlay. Testing = the Test Suite: is_test_suite metadata before init, then launchTestSuite(). Plugin exposes a testSuiteEnabled? option.
  • Revenue is client-side. A global LevelPlayImpressionDataListener fires once per impression, every format, carrying revenue / precision / adNetwork / adUnit / instanceName / placement / country.

Phase 2 — Android rewrite

Delete: appopen/, rewardedinterstitial/, the entire preloading/ package, and ump/.

  • Core (LevelPlayAds.java): replace MobileAds.initialize with LevelPlay.init(context, new LevelPlayInitRequest.Builder(appKey).build(), listener). Hold init state; reject ad calls before onInitSuccess. Drop AdMob md5(ANDROID_ID) test-device logic + the APPLICATION_ID reject gate.

Drop the com.google.android.gms.ads.AdActivity class-name match (never appears under LevelPlay) and apply the bar-hiding to any non-Capacitor activity — interstitial and rewarded ads are always fullscreen, and the mediated network's activity class varies per network.

02

Data flow

Three flows the implementer must get right: the consent → init ordering, the reusable ad lifecycle, and the impression-revenue fan-out.

A · Consent → privacy → init

Strictly ordered. Privacy must reach LevelPlay before init; the consent gate rejects every load* until a decision exists.

B · Ad lifecycle — one reusable object

The old ArrayDeque pool is gone — a loaded LevelPlay ad object is the preloaded ad. Banner follows the same load → ready → show shape.

C · Impression-revenue fan-out

One global listener, registered once at init, covers every format and every winning network — this is the ILRD revenue source.

04

Key code

The three pieces most likely to be done wrong: the consent → init ordering, the replace-all GDPR map, and the once-only revenue listener.

app side · consent must precede init
// 1. consent BEFORE init — the plugin gates ad calls on this
await LevelPlayAds.requestConsentInfo({
  privacyPolicyUrl: 'https://example.com/privacy',
});

// 2. init — app key only, no testMode flag
await LevelPlayAds.initialize({ appKey: '85460dcd' });

// 3. now ad calls are unblocked
await LevelPlayAds.loadInterstitial({ adUnitId: 'abc123' });
if ((await LevelPlayAds.isInterstitialReady()).ready) {
  await LevelPlayAds.showInterstitial();
}
android/.../PrivacyExecutor.java · replace-all GDPR map (Risk #1)
import com.ironsource.mediationsdk.utils.LevelPlayPrivacySettings;

// setGDPRConsents REPLACES the whole map every call —
// the executor owns the canonical map and re-sends it in full.
class PrivacyExecutor {
  private final Map<String,Boolean> gdpr = new HashMap<>();

  void setUserConsent(boolean granted, List<String> networks) {
    for (String n : networks) gdpr.put(n, granted);
    LevelPlayPrivacySettings.setGDPRConsents(gdpr); // full map
  }
  void setDataProcessingOptions(boolean doNotSell) {
    LevelPlayPrivacySettings.setCCPA(doNotSell);
  }
  void setChildDirected(boolean enabled) {
    LevelPlayPrivacySettings.setCOPPA(enabled);
  }
}
android/.../LevelPlayAds.java · register the revenue listener once
// register ONCE, right after LevelPlay.init —
// fires for every format and every winning network
LevelPlay.addImpressionDataListener(data -> {
  JSObject ev = new JSObject();
  ev.put("revenue",      data.getRevenue());
  ev.put("adNetwork",    data.getAdNetwork());   // who won
  ev.put("adUnit",       data.getAdUnit());
  ev.put("precision",    data.getPrecision());
  ev.put("country",      data.getCountry());
  notifyPluginListeners("onAdRevenue", ev);
});

Mitigation

HIGH
PrivacyExecutor owns the canonical map and always re-sends the full set (see §04). The class is com.ironsource.mediationsdk.utils.LevelPlayPrivacySettings — verify against the resolved AAR at the start of M2.
Custom modal vs. per-network CMP policy — LevelPlay itself accepts the modal's boolean/map consent and serves personalized ads (no IAB TCF CMP required). But Google's admob network requires a Google-certified CMP in the EEA/UK.
MED
Fine for LevelPlay core + most networks — and admob is not a default network. If a consumer adds admob to levelplay.networks, the ConsentProvider interface lets them swap in a Google-certified CMP for that case — no API change. Documented in the README, not enforced by the plugin.
